# Multi-Format Enterprise Document Auditor & Data Extractor
[![LinkedIn](https://img.shields.io/badge/Connect_With_Me-LinkedIn-blue?style=for-the-badge&logo=linkedin)](https://www.linkedin.com/in/judith-ngeno-93a28a208/)

An automated, cross-platform compliance processing backend engineered to systematically index, parse, and evaluate unstructured data formats (PDF, DOCX, TXT, CSV) utilizing custom Retrieval-Augmented Generation (RAG) paradigms. This architecture completely detaches view layout updates from structural document calculations, optimizing database tracking speeds.

##  Strategic Product Capabilities
- **Omni-Channel Ingestion Mapping:** Avoids rigid format dependencies by integrating an internal polymorphic router map to link custom data parsing extensions directly to appropriate document processors.
- **Modern LCEL Sequencing:** Bypasses legacy framework chains to ensure lightweight, version-proof processing execution lines.
- **Context-Bound Security Grounding:** Employs zero-temperature model maps and restrictive prompt architecture to eliminate creative hallucinations, ensuring absolute data compliance.
- **Dual-Mode Credential Fallbacks:** Automatically utilizes host configuration secrets in local runtime scopes, dynamically offering secure, in-memory credential input slots when hosted on public portfolio servers.

##  Operational Stack
- **Orchestration Modules:** LangChain Core Components (`langchain-core`, `langchain-openai`)
- **Text Parsers:** PyPDF, python-docx, CSVKit
- **Vector Compute Library:** FAISS (Facebook AI Similarity Search)
- **Interface Layout Framework:** Streamlit Open-Source Library
- **Core Environment:** Python 3.10+
